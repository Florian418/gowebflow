# Getting Started with goWebFlow

## Prerequisites

- Go 1.22+
- Node.js 18+ (only if you use Vite)

---

## 1. Create your project

```
mkdir monsite
cd monsite
go mod init monsite
```

---

## 2. Link goWebFlow locally (go.work)

If goWebFlow is not yet published, use a Go workspace to link both projects:

```
# from the parent folder containing gowebflow/ and monsite/
go work init ./gowebflow ./monsite
```

This tells Go to use your local copy of goWebFlow directly — no need to publish.

## 4. Basic usage (without Vite)

### With layout + pages

**cmd/server/main.go**
```go
package main

import (
    "github.com/Florian418/gowebflow/pkg/httpd"
    "log"
    "net/http"
)

func main() {
    app, err := httpd.New(httpd.Config{
        LayoutDir: "./ui/layouts",
        PageDir:   "./ui/pages",
    })
    if err != nil {
        log.Fatal(err)
    }

    app.Get("/", func(w http.ResponseWriter, r *http.Request) error {
        return app.Render(w, "home.html", nil)
    })

    app.Get("/about", func(w http.ResponseWriter, r *http.Request) error {
        return app.Render(w, "about.html", map[string]any{
            "Title": "À propos",
        })
    })

    log.Println("Server started on http://localhost:8080")
    log.Fatal(app.Listen(":8080"))
}
```

**ui/layouts/base.html**
```html
<!DOCTYPE html>
<html lang="fr">
<head>
    <meta charset="UTF-8">
    <title>Mon site</title>
</head>
<body>
    {{block "content" .}}{{end}}
</body>
</html>
```

**ui/pages/home.html**
```html
{{define "content"}}
<h1>Bienvenue</h1>
{{end}}
```

### Pages only (no layout)

If you don't need a shared layout, omit `LayoutDir`. Each page is rendered as a standalone template:

```go
app, err := httpd.New(httpd.Config{
    PageDir: "./ui/pages",
})
```

```html
<!-- ui/pages/home.html -->
<!DOCTYPE html>
<html lang="fr">
<head><meta charset="UTF-8"><title>Mon site</title></head>
<body>
    <h1>Bienvenue</h1>
</body>
</html>
```

## 5. With Vite

### Install Vite

```
cd ui
npm create vite@latest . -- --template vanilla
npm install
npm install --save-dev cross-env
```

Clean up Vite demo files and set up the multi-theme structure:
```
ui/
├── layouts/
│   └── base.html
├── pages/
│   └── home.html
├── src/
│   └── default/        ← one folder per theme
│       ├── main.js
│       └── style.css
├── dist/               ← generated by build, one subfolder per theme
│   └── default/
├── vite.config.js
└── package.json
```

### Configure Vite

**ui/vite.config.js**
```js
import { defineConfig } from 'vite'
import url from 'node:url'

const tpl = process.env.TPL || 'default'

export default defineConfig(({ command }) => ({
  base: command === 'build' ? '/static/' : '/',
  plugins: [
    {
      name: 'watch-go-templates',
      handleHotUpdate({ file, server }) {
        if (file.endsWith('.html')) {
          server.ws.send({ type: 'full-reload' })
        }
      }
    }
  ],
  server: {
    proxy: {
      '/': {
        target: 'http://localhost:8080',
        changeOrigin: true,
        bypass(req) {
          const u = req.url ?? ''
          if (u.startsWith('/@') || u.startsWith('/node_modules') || u.startsWith('/src/')) {
            return u
          }
        }
      }
    }
  },
  build: {
    manifest: true,
    outDir: `dist/${tpl}`,
    rollupOptions: {
      input: `src/${tpl}/main.js`
    }
  }
}))
```

**ui/package.json scripts**
```json
"scripts": {
  "dev": "vite",
  "build": "vite build",
  "build:default": "cross-env TPL=default vite build",
  "build:all": "npm run build:default"
}
```

**ui/src/default/main.js**
```js
import './style.css'
```

### Update your Go config

```go
app, err := httpd.New(httpd.Config{
    LayoutDir:   "./ui/layouts",
    PageDir:     "./ui/pages",
    StaticDir:   "./ui/dist",   // root — one subfolder per theme
    StaticURL:   "/static/",
    ActiveTheme: "default",
    Dev:         os.Getenv("APP_ENV") != "production",
})
```

`Dev: true` tells gowebflow to use the Vite dev server. `Dev: false` switches to the built manifest (`dist/.vite/manifest.json`).

Using an environment variable avoids hardcoding the value — no code change needed between environments:

```bash
# local dev — APP_ENV unset → Dev: true
go run .

# production
APP_ENV=production go run .
```

### Use the vite template function

**ui/layouts/base.html**
```html
<!DOCTYPE html>
<html lang="fr">
<head>
    <meta charset="UTF-8">
    <title>Mon site</title>
    {{ vite "src/default/main.js" }}
</head>
<body>
    {{block "content" .}}{{end}}
</body>
</html>
```

**ui/pages/home.html**
```html
{{define "content"}}
<h1>Bienvenue</h1>
{{end}}
```

### Adding a new theme

1. Create `ui/src/<name>/main.js` (and `style.css`)
2. Add `"build:<name>": "cross-env TPL=<name> vite build"` to `package.json`
3. Add `&& npm run build:<name>` to `build:all`
4. Use `{{ vite "src/<name>/main.js" }}` in your layout

### Switching theme at runtime

```go
app.Post("/admin/theme", func(w http.ResponseWriter, r *http.Request) error {
    return app.SetTheme(r.FormValue("name"))
})
```

`SetTheme` swaps the active manifest and the static file handler instantly, no restart needed.

## 6. Development workflow (hot reload)

You need two terminals:

**Terminal 1 — Vite dev server (CSS/JS hot reload)**
```
cd ui
npm run dev
```

Vite proxies all non-asset requests to the Go server, so you only open `http://localhost:5173` in your browser.

**Terminal 2 — Go server with air (Go hot reload)**

Install air once:
```
go install github.com/air-verse/air@latest
```

Initialize air config:
```
air init
```

Edit `.air.toml` to point to your main package:
```toml
[build.windows]
  cmd = "go build -o ./tmp/main.exe ./cmd/server"
```

Then start:
```
air
```

With both running:
- Edit a `.go` file → air recompiles and restarts the server
- Edit a `.html` template → browser reloads automatically
- Edit a `.css` or `.js` file → Vite HMR updates without page reload

## 7. Production build

```
cd ui
npm run build:all
```

Then set `Dev: false` in your config and run your Go binary.
The `vite` template function reads hashed filenames from `ui/dist/<theme>/.vite/manifest.json`.

## 9. Error pages

Register custom handlers for 404, 500, or any HTTP code:

```go
// 404 — page non trouvée
app.NotFound(func(w http.ResponseWriter, r *http.Request) error {
    w.WriteHeader(http.StatusNotFound)
    return app.Render(w, "404.html", r.URL.Path)
})

// 500 — masqué en 404 (le client ne sait pas si le serveur a crashé)
app.OnError(http.StatusInternalServerError, func(w http.ResponseWriter, r *http.Request) error {
    w.WriteHeader(http.StatusNotFound)
    return app.Render(w, "404.html", nil)
})

// 403 — masqué en 404 (le client ne sait pas si la ressource existe)
app.OnError(http.StatusForbidden, func(w http.ResponseWriter, r *http.Request) error {
    w.WriteHeader(http.StatusNotFound)
    return app.Render(w, "404.html", nil)
})
```

Trigger a specific status code from a route with `httpd.ErrHTTP`:

```go
app.Get("/admin", func(w http.ResponseWriter, r *http.Request) error {
    if !isAdmin(r) {
        return httpd.ErrHTTP(http.StatusForbidden, fmt.Errorf("forbidden"))
    }
    return app.Render(w, "admin.html", nil)
})
```

`ErrHTTP` signals to the framework which `OnError` handler to call.
The handler decides what the browser receives — including masking the real status code.
